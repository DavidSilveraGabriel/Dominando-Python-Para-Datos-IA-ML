Módulo 2: Programación Orientada a Objetos - Encapsulamiento

Agrupando y protegiendo datos y métodos.

¿Qué es Encapsulamiento?

  1. Agrupación: Datos (atributos) + Métodos juntos en una clase.
  2. Ocultación: Restringir acceso directo al estado interno.

¿Por qué?

  • Protección de datos
  • Abstracción (ocultar complejidad)
  • Flexibilidad y Mantenimiento
  • Control y Validación

Convenciones en Python

nombre

Público (por defecto). Acceso libre.

_nombre

"Protegido" (convención). Uso interno/subclases. No acceder directamente desde fuera.

__nombre

"Privado" (Name Mangling). Evita colisiones/acceso accidental. Se renombra a `_Clase__nombre`.

Getters, Setters y `@property`

Forma Pythonica de controlar acceso a atributos, manteniendo interfaz simple.

class Producto:
    def __init__(self, precio):
        self._precio = max(0, precio) # Atributo "protegido"

    @property # Define el GETTER
    def precio(self):
        """Acceder como: obj.precio"""
        print("(Getter llamado)")
        return self._precio

    @precio.setter # Define el SETTER
    def precio(self, valor):
        """Asignar como: obj.precio = valor"""
        print("(Setter llamado)")
        if valor >= 0:
            self._precio = valor
        else:
            print("Error: Precio negativo.")

    # @precio.deleter # Define el DELETER (opcional)
    # def precio(self): ...

# Uso
p = Producto(100)
print(p.precio)      # Llama al getter -> (Getter llamado) 100
p.precio = 120     # Llama al setter -> (Setter llamado)
print(p.precio)      # Llama al getter -> (Getter llamado) 120
p.precio = -5      # Llama al setter -> (Setter llamado) Error: Precio negativo.

El encapsulamiento protege tus objetos y facilita cambios futuros.
