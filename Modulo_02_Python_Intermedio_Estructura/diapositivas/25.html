Módulo 2: Estructura de un Proyecto Python Estándar

Organizando tu código para claridad y mantenibilidad.

Importancia

  • Claridad y Navegación
  • Mantenibilidad
  • Reutilización (Paquetes)
  • Colaboración
  • Compatibilidad con Herramientas

Estructura Típica

nombre_proyecto/
├── .gitignore
├── README.md
├── requirements.txt
├── pyproject.toml (o setup.py)
│
├── src/               <-- Opción 1
│   └── nombre_paquete/
│       ├── __init__.py
│       └── ... (módulos)
│
├── nombre_paquete/    <-- Opción 2
│   ├── __init__.py
│   └── ... (módulos)
│
├── tests/
│   ├── __init__.py
│   └── test_*.py
│
├── docs/
├── data/ (opcional)
└── scripts/ (opcional)

Opción 1 (`src/`) es recomendada para separar código fuente.

Componentes Clave

.gitignore

Ignora archivos en Git (entornos, logs, pycache).

README.md

Descripción, instalación, uso del proyecto.

requirements.txt

Dependencias para `pip install -r requirements.txt`.

environment.yml

Dependencias para `conda env create -f ...` (opcional).

pyproject.toml

Metadatos y config. de build (moderno, para `pip`).

src/ o nombre_paquete/

Código fuente principal organizado como paquete.

tests/

Pruebas unitarias y de integración.

docs/

Archivos para generar documentación (ej. Sphinx).

data/ (opcional)

Archivos de datos (raw, processed).

scripts/ (opcional)

Scripts auxiliares (procesamiento, reportes).

Adapta la estructura a las necesidades específicas de tu proyecto.
