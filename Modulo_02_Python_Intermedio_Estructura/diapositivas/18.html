Módulo 2: Programación Orientada a Objetos - Herencia

Reutilizando y extendiendo código con jerarquías de clases.

¿Qué es Herencia?

Una clase hija (subclase) hereda atributos y métodos de una clase padre (superclase).

Modela relaciones "Es un/a" (ej. Perro es un Animal).

¿Por Qué Usarla?

  • Reutilización de código (evita duplicar).
  • Modela jerarquías lógicas.
  • Extensibilidad (añadir a la hija sin tocar al padre).
  • Base para Polimorfismo.

Sintaxis y Puntos Clave

class ClasePadre:
    def __init__(self, param_padre):
        self.attr_padre = param_padre
    def metodo_padre(self): ...

class ClaseHija(ClasePadre): # <-- Herencia aquí
    def __init__(self, param_padre, param_hija):
        # 1. Llamar al constructor del padre (¡IMPORTANTE!)
        super().__init__(param_padre) # Recomendado
        # ClasePadre.__init__(self, param_padre) # Alternativa
        self.attr_hija = param_hija # Atributo propio

    def metodo_hija(self): ... # Método propio

    # 2. Sobrescribir método del padre
    def metodo_padre(self):
        print("Versión sobrescrita en la hija")
        # super().metodo_padre() # Opcional: llamar al original
Hijo accede a miembros del padre.
Llamar a `super().__init__()` es crucial si el hijo tiene `__init__`.
Mismo nombre de método en hijo sobrescribe al del padre.
`super().metodo()` llama a la versión del padre.

Veremos un ejemplo práctico de `Animal`, `Perro` y `Gato` a continuación o en el código fuente.
