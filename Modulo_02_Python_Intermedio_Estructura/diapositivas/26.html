Módulo 2: Introducción a las Pruebas Unitarias

Asegurando que tu código funcione como esperas (`unittest` / `pytest`).

¿Por Qué Probar?

  • Detectar errores temprano.
  • Confianza al refactorizar.
  • Documentación viva.
  • Fomenta mejor diseño.
  • Evitar regresiones (errores pasados).

Frameworks

`unittest`

Incorporado, basado en clases (xUnit), `self.assertEqual()`.

`pytest` (Recomendado)

De terceros (`pip install pytest`), sintaxis simple (`assert`), potente, popular.

Ejemplo `unittest`

# test_calculadora_unittest.py
import unittest
from calculadora import sumar

class TestCalculadora(unittest.TestCase):
    def test_sumar_positivos(self):
        # Método empieza con 'test_'
        self.assertEqual(sumar(5, 3), 8)

    def test_sumar_negativos(self):
        self.assertEqual(sumar(-5, -3), -8)

if __name__ == '__main__':
    unittest.main()
Ejecutar:
$ python -m unittest test_calc...py
$ python test_calc...py

Ejemplo `pytest`

# test_calculadora_pytest.py
from calculadora import sumar, restar

# Función empieza con 'test_'
def test_sumar_positivos():
    assert sumar(5, 3) == 8 # Usa assert

def test_sumar_negativos():
    assert sumar(-5, -3) == -8

def test_restar_basico():
    assert restar(10, 4) == 6
Ejecutar:
$ pytest

`pytest` descubre y ejecuta tests automáticamente.

Buenas Prácticas Iniciales

  • Nombres claros (`test_...`)
  • Tests independientes
  • Probar una sola cosa por test
  • Nombres descriptivos
  • Probar casos límite
  • Probar casos erróneos
