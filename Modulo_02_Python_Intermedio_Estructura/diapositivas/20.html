Módulo 2: Módulos y Paquetes

Organizando y reutilizando tu código Python.

Módulos

Un archivo `.py` con código Python (variables, funciones, clases).

Creación:

# mi_modulo.py
VARIABLE = 100
def mi_funcion():
    print("Hola desde módulo")

if __name__ == "__main__":
    # Solo se ejecuta al correr mi_modulo.py
    print("Módulo ejecutado directamente")

Importación (`import`):

# En otro archivo .py
import mi_modulo
print(mi_modulo.VARIABLE) # 100
mi_modulo.mi_funcion()
from mi_modulo import mi_funcion
mi_funcion() # Llamada directa
import mi_modulo as mm # Alias
print(mm.VARIABLE)
# from mi_modulo import * # ¡Mala práctica!

if __name__ == "__main__": evita que código de prueba se ejecute al importar.

Paquetes

Una carpeta con módulos y un archivo `__init__.py` (puede estar vacío) para organizar módulos jerárquicamente.

Estructura Ejemplo:

mi_paquete/
├── __init__.py
├── modulo_a.py
└── sub_paquete/
    ├── __init__.py
    └── modulo_c.py

Importación (Notación de Puntos):

import mi_paquete.modulo_a
mi_paquete.modulo_a.funcion_a()

from mi_paquete.modulo_a import funcion_a
funcion_a()

import mi_paquete.sub_paquete.modulo_c as mod_c
mod_c.funcion_c()

from mi_paquete.sub_paquete import modulo_c
modulo_c.funcion_c()

Rol de `__init__.py`:

  • Marca directorio como paquete.
  • Se ejecuta al importar el paquete.
  • Puede contener código de inicialización.
  • Puede exponer miembros de submódulos (usando `from .modulo import miembro`).
  • Puede definir `__all__` para `import *`.

¡Módulos y paquetes son clave para proyectos escalables!
