Despliegue con Docker: Contenerización

Empaquetando tu aplicación para un despliegue consistente.

¿Qué es Docker y Contenedores?

Contenedor: Unidad estándar que empaqueta código y dependencias.

Imagen Docker: Plantilla para crear contenedores.

Docker: Plataforma para construir y ejecutar contenedores.

Ventajas:

  • Consistencia entre entornos.
  • Aislamiento de aplicaciones.
  • Portabilidad.
  • Eficiencia (más ligeros que VMs).
  • Escalabilidad.
  • Reproducibilidad.

Componentes Clave

Dockerfile: Instrucciones para construir una imagen.

FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["flask", "run", "--host=0.0.0.0"]

Imagen Docker: Plantilla creada del Dockerfile (`docker build`).

Contenedor Docker: Instancia en ejecución de una imagen (`docker run`).

Comandos Útiles:

  • `docker build`
  • `docker run`
  • `docker ps`
  • `docker logs`
  • `docker stop/start/rm`

Docker es fundamental para un despliegue fiable y consistente.
