Introducción a Git y GitHub

Control de versiones esencial para tus proyectos

+

Git

Sistema de control de versiones distribuido que se ejecuta localmente

GitHub

Plataforma web para alojar repositorios Git en la nube

GitHub es tu portafolio como desarrollador - los reclutadores pueden ver tus proyectos y código

Beneficios

Historial Completo

Rastrea quién cambió qué, cuándo y por qué

Revertir Cambios

Vuelve a versiones anteriores si algo sale mal

Experimentación Segura

Usa ramas para trabajar en paralelo sin afectar la versión principal

Colaboración

Facilita que varios desarrolladores trabajen juntos

Conceptos Básicos

  • Repositorio

    Carpeta del proyecto rastreada por Git (contiene subcarpeta .git)

  • Commit

    Instantánea de cambios guardada con un mensaje descriptivo

  • Branch (Rama)

    Línea independiente de desarrollo. La principal es main

  • Merge (Fusión)

    Combinar los cambios de una rama en otra

  • Clone (Clonar)

    Copiar un repositorio remoto a tu máquina local

  • Push (Empujar)

    Enviar tus commits al repositorio remoto

  • Pull (Jalar)

    Traer cambios desde el repositorio remoto al local

  • Add (Añadir)

    Marcar archivos para el próximo commit (staging area)

Instalación y Configuración

Instalar Git

  1. Visita git-scm.com/downloads
  2. Descarga para tu sistema operativo
  3. Ejecuta el instalador (opciones por defecto)
  4. Verifica con git --version

GitHub

  1. Crea cuenta en github.com
  2. Configura tu perfil
  3. Crea tu primer repositorio
  4. Conecta Git local usando HTTPS o SSH

Configuración Básica

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Importancia para el curso: Usa Git y GitHub desde el principio para guardar tu trabajo y construir tu portafolio.
