Gestión de Entornos Virtuales

Aislamiento y organización de dependencias para tus proyectos Python

¿Qué es un Entorno Virtual?

Un directorio aislado que contiene:

  • Una instalación específica de Python
  • Conjunto independiente de librerías
  • Aislado de otros proyectos y del sistema

¿Por qué son esenciales?

Evita conflictos entre dependencias de diferentes proyectos

Facilita la reproducibilidad de tus proyectos

Mantiene tu sistema limpio y organizado

Herramientas Estándar: venv y pip

venv

  • Módulo estándar en Python 3.3+
  • Crea entornos virtuales ligeros
  • Usa el intérprete Python con el que se creó

pip

  • Gestor de paquetes estándar para Python
  • Instala, actualiza y desinstala librerías
  • Se instala automáticamente con venv

Comandos Esenciales

Crear Entorno

$ python -m venv venv

Crea una carpeta "venv" con todo lo necesario para el entorno virtual

Activar Entorno

venv\Scripts\activate
source venv/bin/activate

Instalar Paquetes

(venv) $ pip install numpy pandas
(venv) $ pip install -r requirements.txt

Listar y Guardar

(venv) $ pip list
(venv) $ pip freeze > requirements.txt

Desactivar

(venv) $ deactivate

Vuelve al entorno Python del sistema

Alternativa: conda

conda - Gestor de paquetes y entornos que viene con Anaconda/Miniconda

Ventaja: maneja dependencias de Python y no-Python (C/C++, R)

Recomendación para el curso

¡Crea y activa SIEMPRE un entorno virtual para cada nuevo proyecto Python! Esto te ahorrará muchos problemas de dependencias en el futuro.

Problema de dependencias sin entornos virtuales

Proyecto A requiere SuperLib v1.0 ⚔️ Proyecto B requiere SuperLib v2.5
Con entornos virtuales: venv-A con SuperLib v1.0 + venv-B con SuperLib v2.5
